## 1. 한의학 공개 데이터 소스

### 1.1 한의학고전DB (mediclassics.kr)

**운영 주체:** 한국한의학연구원(KIOM)

**제공 데이터:**
- 총 161종 한의학 고전 서적 수록
- 약 1,816만 자(18,162K) 한문 원문
- 주요 수록 서적: 동의보감, 황제내경, 본초강목, 태평혜민화제국방, 방약합편, 침구경험방, 석곡심서 등
- 한문 원문 + 한국어 번역 동시 제공
- 회원가입 없이 누구나 열람·검색 가능

**접근 방법:**
- 웹 인터페이스: https://mediclassics.kr/
- 고의서 원문 배포 서비스: http://info.mediclassics.kr/apps/dist-texts/index.html
- 모바일 앱 지원

**라이선스:** Creative Commons Attribution-NonCommercial 4.0 International (CC BY-NC 4.0)
- 비영리 목적: 자유 사용 가능, 출처 표시 필수
- 영리 목적: kiombook@kiom.re.kr로 별도 협의 필요
- 재배포 가능 (단, 주기적 업데이트 반영 권고)

**특이사항:**
- 2024년 서비스 중단 위기 후 재개됨 (한의계 강력 반대로 유지)
- 동의보감AI™ 베타 서비스 제공 중 (자체 AI 기능 탑재)
- API 미제공 — 웹 크롤링 또는 원문 배포 서비스 활용 필요

## 3. 기존 한의학 AI/디지털 도구 분석

### 3.1 국내 서비스

#### 동의보감AI™ (KIOM/Mediclassics)
- **유형:** 경전 기반 검색 AI
- **기능:** 동의보감 원문 기반 자연어 검색 및 질의응답 (베타)
- **강점:** 공신력 있는 원전 데이터 기반, 국가기관 운영
- **약점:** 베타 단계, 처방 추천/변증 기능 없음, 현대 임상 적용 한계

#### 준차트 (AI 빅데이터 기반 전자차트)
- **유형:** 한의원 전자차트 + AI 분석
- **기능:** 한의 임상 데이터 수집 및 빅데이터 기반 분석, 한의학 세계화 목표
- **강점:** 실제 임상 데이터 축적 가능
- **약점:** 서비스 초기 단계, 데이터 축적량 미흡

#### 한의약인(人)AI
- **유형:** 빅데이터 기반 전자차트/임상 지원
- **운영:** 한국한의약진흥원(NIKOM)
- **기능:** 한의 임상 정보 AI 분석 보조
- **현황:** 공공기관 주도 초기 개발 단계

## 4. 한의학 AI 특수 고려사항

### 4.1 법적 규제

#### 식약처 AI 의료기기 분류 기준 (2024~2025)
AI 진단 보조 소프트웨어가 의료기기에 해당하는지 판단 기준:
1. 소프트웨어가 의도대로 작동하지 않을 때 환자에게 위해를 끼칠 가능성
2. 의료인의 임상적 판단을 보장하는지 여부

**허가 심사 요건:**
- 학습데이터 특성 및 업데이트 주기
- 민감도·특이도·AUC 등 진단 정확성
- 시험데이터셋 독립성·객관성

**2024년 세계 최초:** 식약처 생성형 AI 의료기기 허가·심사 가이드라인 공개

#### 한의학 진단 AI의 법적 지위
- **진단 보조** 도구 (의료인 최종 판단 보장): 의료기기 허가 필요
- **정보 제공** 도구 (단순 검색/교육): 의료기기 미해당 가능성
- **처방 추천** 기능: 의료행위에 해당할 수 있어 높은 규제 위험

**권고:** RAG 기반 정보 검색·요약 도구로 포지셔닝 → 의료기기 규제 회피 가능성 높음. "보조 도구이며 최종 판단은 한의사가 한다"는 면책 조건 필수.

### 4.2 한의학 용어 표준화 문제

#### 한국표준질병사인분류 (KCD) 한의 분야
- KCD 7차 개정(2016년 시행)에서 한의 분류를 **U코드(특수목적코드)**로 통합
- 한의병명, 한의변증, 사상체질병증 등 포함
- KCD 코드와 전통 변증 체계 간 매핑 불완전

#### 주요 표준화 문제점
1. **동일 증상 다중 변증**: 같은 임상 증상이 팔강변증, 장부변증, 육경변증에서 각기 다르게 분류됨
2. **한자 이체자 문제**: 경전 원문의 동일 개념이 여러 한자 표기로 혼재
3. **시대별 개념 변화**: 고전 용어와 현대 임상 용어 간 의미 편차
4. **번역 표준 부재**: 한의학 영문 표준 용어(WHO 기준)와 국내 실용 표기 불일치

---

### 4.3 변증-처방 매핑의 복잡성

#### 구조적 특성
```
증상(症狀)
  ↓ [변증(辨證)]
병증(病證) — 팔강변증 / 장부변증 / 육경변증 / 위기영혈변증
  ↓ [치법(治法)]
처방(處方) — 1:N 매핑 (동일 변증에 다수 처방 존재)
  ↓ [가감(加減)]
개인화 처방 — 환자 체질·현증·병력 반영
```

#### 핵심 난제
- **동병이치(同病異治):** 같은 질환도 변증에 따라 처방이 달라짐
- **이병동치(異病同治):** 다른 질환이라도 변증이 같으면 동일 처방 가능
- **가감의 맥락의존성:** 처방 가감이 환자의 개별 상황에 크게 의존
- **정량화 한계:** 변증 판단은 사진(四診: 망·문·문·절) 종합으로 텍스트화 어려움

### 5.2 한의학 지식의 계층 구조 및 청크 설계

#### 추천 문서 청크 단위

```

```

### 5.3 RAG 파이프라인 설계 권고

#### 1단계: 하이브리드 검색
- **Dense retrieval:** 증상/변증 의미 유사도 검색 (한국어 임베딩 모델)
- **Sparse retrieval (BM25):** 한자 용어 정확 매칭
- **이유:** 한의학 용어는 동의어·이체자 많아 의미 기반 + 키워드 기반 병행 필수

#### 2단계: 재순위화 (Reranking)
- 검색된 문서를 맥락(변증 단계 vs 처방 단계) 기준으로 재정렬
- 환자 증상 입력 → 변증 관련 청크 우선, 이후 처방 청크 연계

#### 3단계: 컨텍스트 주입
- **원전 + 현대 해석 동시 주입:** 고전 텍스트만으로는 현대 한의사 이해도 낮음
- **근거 수준 표시:** 경전 기반 / 임상 연구 기반 / 전문가 합의 구분

#### 4단계: 생성 제약
- 처방 추천 시 반드시 "최종 판단은 한의사에게" 문구 자동 삽입
- 용량, 금기 관련 답변 시 경고 문구 포함

### 5.4 한국어 임베딩 모델 권고

## 6. 종합 평가 및 권고사항

### 6.1 데이터 소싱 전략 요약

**단기 (즉시 활용 가능):**
- 한의학고전DB 원문 배포 서비스 → 경전 161종 CC BY-NC 활용
- OASIS 약재백과 478건 → 본초 데이터 기반
- KMCRIC 표준경혈DB → 경혈 360+ 기본 데이터

**중기 (협의 필요):**
- KIOM과 상업 라이선스 협의 → 번역본·구조화 데이터 활용
- 한의원 전자차트 데이터 제공 기관과 파트너십
- AI-Hub 데이터셋 신청·활용

**장기 (자체 구축):**
- 실제 한의사 참여 변증-처방 데이터 라벨링
- 방제학 교재 기반 구조화 처방 DB 구축 (저작권 확인 필수)

### 6.2 차별화 포인트

기존 서비스(동의보감AI, 준차트 등) 대비 차별화 방향:
1. **변증-처방 연계 RAG**: 증상 입력 → 변증 추론 → 처방 근거 제시 (단계별 논거 추적 가능)
2. **원전 근거 추적**: 모든 답변에 원전 출처 (동의보감 ○권 ○편) 표시
3. **현대 임상 근거 병기**: 경전 + 최신 RCT/논문 동시 참조
4. **한의사 워크플로우 통합**: 진료 흐름(주소증 → 변증 → 처방 → 가감)에 맞는 UX

|---|---|---|
